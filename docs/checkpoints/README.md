# Checkpoints (Detailed)

This folder contains detailed, implementable guidance for each development checkpoint. Start with the summary in `../checkpoints.md`, then dive into the files below.

- 00-bootstrap.md – Project initialization, dependencies, migrations layout
- 01-config-logging.md – Configuration system and logging
- 02-db-sessions.md – SQLx setup, migrations, session store
- 03-templates-ui.md – Askama templates, Pico.css, htmx
- 04-auth-login-logout.md – Argon2id, CSRF, login/logout
- 05-upload.md – Multipart streaming, size limits, storage paths
- 06-file-details-code-lookup.md – Code lookup, file details page
- 07-direct-link.md – Signed token generation and validation
- 08-settings-admin.md – Admin-only settings UI and persistence
- 09-cleanup-job.md – Background task for expirations
- 10-rate-limiting-hardening.md – Governor policies and headers
- 11-docker-deploy.md – Containerization and deployment checks
- 12-qa-tests.md – Manual and automated verification
